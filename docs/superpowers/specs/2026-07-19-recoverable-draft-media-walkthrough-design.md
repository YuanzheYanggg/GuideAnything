# 可恢复草稿与无标记图片讲解设计

## 目标

让指南编辑在误删含标注图片后可以可靠恢复，并让图片教程在学习模式中以居中放大的镜头引导替代遮挡画面的编号和高亮标记。

## 范围

- 每份指南保存最近 200 个服务器端草稿快照。
- 编辑者可查看草稿历史，并把任一历史草稿恢复为新的当前 revision。
- 删除包含至少一条图片标注的节点前必须显式确认；节点右上角按钮、工具栏删除和键盘删除共用该保护。
- 图片标注编辑时继续显示编号点和矩形，作为作者指定讲解位置的辅助。
- 学习模式不再渲染任何图片标记或遮罩，只使用标注的镜头或由标注区域推导出的镜头来居中、放大图片。

不在范围内：删除媒体文件、为每个普通节点增加确认、为历史版本实现视觉 diff、改变已发布版本或改变知识检索结果。

## 现状与问题

当前 `guides.draft_document` 只保存最新草稿。虽然每次保存都会递增 `revision`，但旧草稿会被覆盖；编辑器内存中只有当前页面生命周期内的 80 步撤销。图片标注位于图片节点的 `data.annotations`，因此删除图片节点会同时移除全部标注和关联边。学习播放器会把所有标注渲染为可点击的编号标记，造成截图遮挡。

## 方案选择

采用服务器端不可变草稿快照。浏览器内撤销不能跨刷新或设备恢复；把每次草稿保存作为发布版本会污染面向学习者的正式版本序列。草稿快照独立于发布版本，适合自动保存和误删恢复。

## 数据与后端

新增迁移和 `guide_draft_revisions` 表。每行保存：快照 ID、指南 ID、对应 revision、标题、摘要、标签、完整 `CanvasDocument`、保存人和保存时间。`(guide_id, revision)` 唯一，快照记录不可更新。

每次 `PATCH /api/guides/:id` 成功更新当前草稿后，在同一事务中写入该新 revision 的快照；任一步失败则整次保存回滚。写入后按该指南的时间和 revision 删除超过 200 条的最旧草稿快照。`guide_versions` 继续只存发布版本，绝不参与这个清理。

新增两个受现有草稿编辑权限保护的接口：

- `GET /api/guides/:id/draft-history`：返回历史列表的元数据，不向列表响应重复发送完整画布。
- `POST /api/guides/:id/draft-history/:revision/restore`：请求体携带当前 revision。服务端读取目标快照，以当前 revision 为条件创建一个新的草稿 revision 和快照；目标快照保持不变。

恢复和常规保存都复用现有的 revision 冲突语义以及草稿流程索引同步。若目标快照已因 200 条上限被清理、无编辑权限或当前 revision 已变化，恢复失败且不改动当前草稿。

## 编辑器交互

编辑器页头增加“草稿历史”入口。历史面板按最新优先显示 revision、保存时间和保存人；当前 revision 标记为“当前版本”。每个非当前条目提供“恢复此版”。恢复操作需单独确认，文案明确说明它会创建一个新的当前草稿，不会删除现有历史。

节点删除改为集中判断。若待删除节点中有图片节点且 `annotations.length > 0`，打开确认对话框，显示将删除的图片数量和标注数量，说明图片文件本身不会被删除且可从草稿历史恢复。取消不改变画布；确认后执行原有的节点、边、步骤和入口/出口清理。普通节点、空标注图片和边保持现有直接删除体验。

## 学习模式的图片讲解

`ImageAnnotationPlayer` 保留开始讲解、自动播放、上一步/下一步、右侧讲解文字和关联资料按钮。播放器移除图片上的标记按钮及其样式，不再以点击编号跳转。右侧计数使用“讲解 N / 总数”。

当前讲解使用 `cameraForAnnotation`：有已保存镜头时使用该镜头；旧标注没有镜头时，点标注以该点为中心、默认放大 2.5 倍，矩形标注以矩形中心为镜头中心并按区域大小推导缩放。因而旧标注不需要数据迁移或人工重做，作者也只需大致指出位置。

## 权限、安全与失败行为

草稿历史与恢复遵循现有指南 OWNER/EDIT 权限；学习者不能读取或恢复未发布草稿。历史画布仅由恢复接口在服务端读取，列表 API 不暴露整份文档。所有数据库写入采用同一事务，并继续通过 `CanvasDocumentSchema` 验证文档。恢复产生新 revision，防止旧客户端静默覆盖并使流程索引可重新对齐。

## 验收与测试

- API 测试：保存生成快照；超过 200 条时仅删除该指南最旧的快照；无权限、缺失快照和过期 revision 被拒绝；恢复生成新 revision、保留来源快照并同步当前文档。
- 编辑器测试：从三个删除入口删除带标注图片都会先出现确认；取消不保存；确认后删除并可在历史中恢复。
- 播放器测试：开始、切换和自动播放仍改变镜头和右侧讲解；图片内不再出现“播放标注”按钮或标记；无显式镜头的旧点、框仍产生正确的居中缩放。
- 回归：现有发布版本读取、草稿保存、流程索引和图片标注编辑测试保持通过。
