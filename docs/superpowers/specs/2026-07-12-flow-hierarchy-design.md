# GuideAnything 业务流程层级体验设计

## 1. 目标与问题

GuideAnything 的画布当前把流程节点、Markdown、图片和视频都视为同一平面元素；`自动布局`只会放入固定三列网格。即使作者已经连接了主流程，图片和说明仍会与流程骨架争夺视觉位置。学习模式又把 `steps[]` 作为独立的线性列表，无法向学习者说明某份资料属于哪个业务动作或阶段。

本次改造把业务流程设为唯一的一级结构：

```text
业务阶段（可选泳道）
  └─ 主流程节点（方向、分支、子指南）
       └─ 挂靠资料（Markdown、图片、视频）
```

**成功标准**：作者能先画主流程、再把多模态资料挂在具体动作上；一次自动整理可以形成可解释、无重叠的左到右结构；学习者沿流程学习时只看到当前节点有关的资料，而不会被所有媒体节点淹没。

## 2. 用户与核心路径

### 流程梳理者（作者/编辑者）

1. 先添加开始、动作、判断、数据、结束和子指南，连接主流程。
2. 可为主流程节点标记业务阶段，例如“准备”“录入”“校验”“完成”。
3. 选中一个主流程节点后添加 Markdown、图片或视频，资料默认挂靠该节点；也可在属性面板重新挂靠或脱离。
4. 在左侧结构面板检查“阶段 → 主流程 → 挂靠资料”的树，定位未挂靠资料。
5. 点击“自动整理”，先看到布局摘要和预览；确认后应用布局，取消不会改变草稿。
6. 发布仍保存同一份 `CanvasDocument`，固定版本子指南仍保持不可变引用语义。

### 流程学习者

1. 看到当前学习步骤所在的业务阶段，以及该步骤在阶段中的位置。
2. 画布焦点始终落在当前主流程节点，流程方向和分支可辨。
3. 在“本步骤资料”中查看挂靠的说明、截图和视频；这些资料默认不额外占用步骤序号。
4. 作者显式把资料节点加入教学步骤时，仍按既有规则保留为独立步骤，避免破坏已有发布版本。

## 3. 方案比较

1. **业务流程为骨架，资料作为挂靠层（采用）**：保留现有 React Flow 与文档模型，语义清晰，自动布局能优先保证箭头和分支可读；学习内容跟随具体操作出现。
2. **按教学章节分区画布**：适合课程大纲，但同一业务分支会跨章节，无法优先回答“下一步做什么”。
3. **保留平面画布，只新增自动网格**：实现最少，却不能表达资料归属，也无法让学习模式读取真正的层级。

## 4. 信息模型与兼容性

### 4.1 文档字段

保持 `schemaVersion: 1`，为 JSON 文档增加全可选字段，因此旧草稿和旧发布版本无需迁移：

```ts
type FlowStage = {
  id: string;
  title: string;
  order: number;
  description?: string;
};

type CanvasNodeBase = {
  // 既有字段…
  stageId?: string;
  contentParentId?: string;
};

type CanvasDocument = {
  // 既有字段…
  stages?: FlowStage[];
};
```

### 4.2 约束

- 只有 `markdown`、`image`、`video` 三类节点可以拥有 `contentParentId`。
- `contentParentId` 必须引用同一文档中存在的一级主流程节点（开始、结束、流程、判断、数据或子指南），不能引用自身、资料节点或展开产物。
- `stageId` 只能标记一级主流程节点，且必须引用 `stages` 中的阶段；挂靠资料继承主节点阶段，未标记的主节点属于“未分阶段”。
- 资料仍可有常规连线以兼容已有画布，但自动布局只把主流程间的可见连线当作骨架；资料的归属由 `contentParentId` 表达，不以连线猜测。
- 子指南根节点属于主流程节点；展开产物沿用引用节点的阶段，在布局和结构面板中以“子指南内容”缩进展示，不能被单独挂靠。

### 4.3 兼容策略

缺失 `stages`、`stageId` 或 `contentParentId` 的旧文档照常读取。旧资料节点显示在结构面板的“未挂靠资料”，不会自动改写业务内容。作者确认自动整理或显式重新挂靠后才会保存新字段。发布、搜索、版本和子指南展开/折叠的现有语义不变。

## 5. 画布交互与视觉层级

### 5.1 编辑器布局

编辑器由“左结构面板 + 中央画布 + 右属性面板”组成；左面板可折叠，窄屏沿用现有抽屉降级。

- **结构面板**：按阶段列出主流程；每个主流程节点下列出挂靠资料及数量。点击任何项目会选中并聚焦画布；未挂靠资料单独列出，提供明确整理入口。
- **画布**：主流程卡片保持完整边框和较高对比；挂靠资料使用较轻的资料卡样式、虚线层级连线和“说明 / 截图 / 视频”标识，始终围绕所属动作排列。阶段以低饱和泳道背景和阶段标题标识，而不是作为可连接的假节点。
- **属性面板**：主流程节点可编辑阶段并看到“已挂靠 N 条资料”；资料节点显示“挂靠到流程节点”选择器、脱离按钮和“加入教学步骤”按钮。
- **添加行为**：当前选中主流程时新增资料自动设置 `contentParentId`；未选中主流程时创建未挂靠资料，并在保存前提示作者可选择整理。

### 5.2 自动整理（替换固定网格）

工具栏把“自动布局”升级为“自动整理”。操作分为计算预览与应用两步：

1. 读取可见主流程节点和主流程连线；资料节点、隐藏节点与折叠子指南内部节点不参与骨架排名。
2. 以 `entryNodeId` 为优先入口，缺失时采用无入边节点，再以稳定的现有位置和 ID 作为并列排序，计算从左至右的层级 rank。
3. 每个 `(阶段, rank)` 单元纵向排列主流程节点；判断节点的出边标签决定稳定的“是 / 否”分支顺序，其余分支按原有位置和 ID 排序。
4. 每个资料节点在所属主流程节点的详情轨道中自上而下排列；资料不再占用主流程 rank。无主节点的资料置于“待整理”区域且不会与主流程重叠。
5. 阶段作为从上到下堆叠的横向泳道，按 `stage.order` 排列；未分阶段泳道总在最后。布局计算阶段边界，供前端渲染背景框。
6. 对循环或无入口的流程，算法保留每个节点、输出诊断并以稳定顺序补齐 rank；不会删除边、节点、步骤或引用产物。

预览条必须列出：主流程节点数、阶段数、已挂靠资料数、未挂靠资料数、未连通主流程节点数、循环诊断数。作者仅在点击“应用布局”后写入历史栈和自动保存。

### 5.3 可解释性与可访问性

- 每次预览以可读文本说明布局规则，例如“按入口从左到右排列；3 条资料挂靠到对应操作；1 条资料仍未归类”。
- 结构面板使用可键盘操作的树形列表，当前选中项具有可见焦点，折叠/展开有明确 `aria-label` 与状态。
- 预览、应用、取消和未挂靠提醒均通过 `aria-live` 给出状态；颜色不作为阶段、分支或附件归属的唯一信息。

## 6. 学习模式

- 步骤侧栏按阶段添加非交互分组标题；资料的显式教学步骤继承其主节点阶段，未分阶段步骤归入“未分阶段”。
- 右侧内容在当前步骤下渲染 `contentParentId === currentStep.nodeId` 的资料，标题为“本步骤资料”。资料按创建时的画布顺序稳定排列。
- Markdown、图片、视频沿用既有安全渲染和关键点行为；视频关键点仍可跳转。
- 画布继续聚焦当前步骤节点。资料卡仅作为当前节点的辅助内容显示，不强迫学习者在画布中逐一选择。
- 没有步骤的发布版本仍显示既有空状态；不在本次范围内自动生成并发布教学步骤。

## 7. 模块边界

| 模块 | 责任 |
| --- | --- |
| `packages/contracts` | `FlowStage`、节点归属字段及跨字段校验；旧文档兼容。 |
| `packages/canvas-core` | 主流程层级布局、诊断和阶段边界的无副作用算法。 |
| `features/editor` | 资料挂靠、阶段编辑、结构面板、可撤销布局预览与应用。 |
| `features/lesson` | 阶段分组和当前步骤资料聚合。 |
| `styles.css` | 主流程、资料、泳道、结构树和响应式布局的视觉层级。 |

## 8. 验收与测试

1. 共享 schema 拒绝无效阶段、无效资料归属、资料嵌套和引用展开产物挂靠；旧文档仍可通过。
2. 纯算法测试证明：线性流、判断分支、阶段、挂靠资料、未挂靠资料、循环和不连通分量均稳定且不重叠；1000 节点基准保持快速。
3. 编辑器组件测试证明：选中主流程新增资料会自动挂靠；可重挂靠/脱离；结构树可选择节点；预览不会持久化，应用会进入撤销历史。
4. 学习组件测试证明：步骤按阶段显示，当前节点资料被聚合，显式资料步骤仍按原有顺序可学习。
5. 浏览器验收使用种子“ERP 销售订单创建”：整理前后可看见主流程、分支、资料与子指南的层级关系；学习者能沿业务流程阅读并查看当前步骤资料。
6. 最终执行 `pnpm lint && pnpm typecheck && pnpm test && pnpm build`，并检查真实运行时控制台错误与关键交互。

## 9. 明确不做与后续机会

本次不新增实时协作、自动保存布局历史、AI 生成流程、外部媒体源、数据库迁移或改变发布快照语义。

在这一层级基础稳定后，下一阶段可评估：分支条件驱动的自适应学习路径、可复用阶段模板、流程健康检查（无入口/无出口/孤立资料）、章节级进度和批注协作。这些不是当前实现的前置条件。
