# 编辑器纵向责任泳道视觉设计

## 目标

在 GuideAnything 编辑器的无限画布中增加纵向责任泳道，让拥有相同 `laneId` 的节点在视觉上形成同一列，同时保持泳道的存在感低于节点、连线和业务阶段。

## 当前实现依据

- `CanvasDocument.lanes` 保存责任泳道定义。
- 主流程节点通过 `laneId` 关联泳道。
- `@guideanything/canvas-core` 已提供 `getSwimlaneBounds(document)`，其几何结果会随现有节点位置、阶段和泳道配置变化。
- 编辑器已使用 `ViewportPortal` 渲染跟随画布缩放/平移的阶段背景。
- 学习页已有泳道视觉结构，可复用其语义方向，但编辑器需要更低对比度的版本。

## 设计方案

### 数据与渲染

编辑器从当前 `renderedDocument` 计算 `getSwimlaneBounds(renderedDocument)`，交给独立的 `CanvasSwimlanes` 展示组件。组件只输出装饰性 DOM，不更新文档、不改变节点位置、不参与拖拽和连线事件。

每个 bounds 输出一个 `.canvas-swimlane`：

- `data-lane-id` 保留真实泳道 ID，未分配泳道使用 `unassigned`；
- 顶部显示泳道名称和“责任 / 系统 / 未分配”轻量徽标；
- `aria-hidden="true"`，避免装饰层进入键盘阅读顺序；
- `pointer-events: none`，确保节点、画布和连线继续接收原有事件。

### 视觉层级

- 泳道使用低透明度深色底、细边界和轻微纵向渐变；
- 角色泳道采用冷青蓝色，系统泳道采用低饱和琥珀色，未分配泳道采用中性灰蓝色；
- 业务阶段作为横向辅助层继续存在，两个维度通过透明度和边框区分，不互相遮挡；
- 泳道标签只在顶部出现，不在每个节点上重复显示；节点底部已有责任信息继续保留，作为局部确认。

### 响应式与可访问性

- 泳道和节点共享 React Flow viewport，因此缩放、平移和自动整理后仍保持对齐；
- 小屏下缩小标签间距与徽标字号，不改变几何计算；
- 泳道不添加动画，减少画布持续运动和视觉噪音；减少动效或透明度模式下继续使用不透明边界和纯色底。

## 验收标准

1. 有配置泳道时，编辑器 DOM 中每条可见泳道都有一个 `data-testid="canvas-swimlane"`。
2. 同一 `laneId` 的节点处于同一泳道 bounds 横向范围内；未分配节点只出现在“未分配责任”区域。
3. 泳道背景可被看出，但不会盖住节点、连线、节点 Handles 或画布交互。
4. 画布缩放、平移、节点拖动和自动整理不改变泳道数据，也不引入新的持久化字段。
5. 现有 Web 测试、类型检查、构建和真实浏览器交互继续通过。
