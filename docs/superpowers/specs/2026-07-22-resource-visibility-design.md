# 资料隐藏与恢复设计

## 目标

为画布中的 Markdown、图片和视频资料增加小眼睛按钮。作者可以在编辑画布中隐藏或恢复一份资料；隐藏状态持久化到指南文档，发布后的学习页、流程资料展示、图片/视频跳转和工作区 AI 知识索引都不应继续使用该资料。

作者仍然可以在编辑画布和流程结构面板中看到隐藏资料，并通过斜眼按钮恢复。隐藏不是删除，媒体资产、资料内容、挂靠关系、标注和历史草稿都必须保留。

## 方案选择

### 方案 A：复用现有 `hidden` 字段

优点是改动少，学习页已有部分 `hidden` 过滤逻辑。缺点是该字段同时承担子指南派生节点、隐藏连线和布局计算语义；如果资料也写入该字段，作者画布可能直接移除资料卡，恢复入口会变得不稳定，并且会把结构隐藏和产品可见性混为一谈。

### 方案 B：新增资料专用 `visibility` 字段（采用）

在 Canvas 资料节点上新增可选的 `visibility: 'VISIBLE' | 'HIDDEN'`。字段缺失等同于 `VISIBLE`，兼容所有已有草稿和已发布版本。作者端不把该字段映射为 React Flow 的结构 `hidden`，因此隐藏资料仍留在画布和层级面板中；学习页和知识快照编译器分别按该字段过滤。

优点是数据语义清晰、恢复路径稳定、不会影响子指南展开和布局算法。代价是需要同时更新协议、编辑器、学习页和 V1/V2 知识投影测试。

### 方案 C：仅在前端临时隐藏

只在 React 状态中切换资料卡显示，不写入指南文档。该方案无法跨刷新、发布、学习页和 AI 检索生效，不满足需求。

## 交互设计

- 仅 Markdown、图片和视频节点显示可见性按钮；流程节点和子指南节点不显示。
- 按钮放在现有删除按钮旁边，使用 Phosphor 的 `Eye` / `EyeSlash` 图标。
- 可见状态按钮的无障碍名称为“隐藏资料”；隐藏状态按钮的无障碍名称为“显示资料”，并用 `aria-pressed` 表达当前隐藏状态。
- 点击后立即更新当前文档、历史栈和未保存状态；保存草稿或发布沿用既有持久化流程。
- 隐藏资料在作者画布中保留节点和内容，但使用弱化视觉状态并显示“已隐藏”提示；仍可从资料节点或结构面板选中并恢复。
- 资料附录分组和挂靠关系不因隐藏状态被删除或重新排序。

## 数据流与过滤边界

### Canvas 协议

- `CanvasNode` 的基础字段增加可选 `visibility`。
- `CanvasDocumentSchema` 允许字段缺失并视为可见；只有 Markdown、图片、视频资料节点可以拥有该字段。
- 不改变现有 `hidden` 字段及其子指南、连线和布局语义。

### 作者编辑器

- `toFlowNodes` 将资料可见性作为节点展示数据传给资料节点组件，但不转换为 React Flow 的 `hidden` 属性。
- `NodeActionProvider` 扩展资料可见性切换回调；`NodeChrome` 只在资料节点上渲染按钮。
- 切换逻辑使用 `HIDDEN` 与字段缺失/`VISIBLE` 之间的可逆转换，保留其余节点数据不变。

### 发布学习页

- 教学步骤中的隐藏资料不出现在资料列表和当前步骤内容中。
- 学习流程图对隐藏资料节点及其关联连线隐藏，但不改变作者保存的文档结构。
- 图片标注、视频关键点、资料引用和直接定位不得打开隐藏资料；若目标已隐藏，跳转入口失效但不导致学习页崩溃。

### 知识索引与流程摘要

- V1 和 V2 流程知识编译器都不把隐藏资料投影到资源列表。
- 隐藏资料不产生 `USES_RESOURCE`、`RESOURCE_REFERENCE` 或资料教学步骤；指向隐藏资料的既有关系被忽略，不把作者主动隐藏误报为结构损坏。
- 现有保存/发布索引同步机制负责按新文档校验和重新物化索引，因此旧资料片段会随新的流程快照被替换。

## 错误处理与兼容性

- 旧文档没有 `visibility` 时按可见处理。
- 隐藏资料仍保留在草稿历史和已发布文档 JSON 中，允许作者恢复后再次发布。
- 隐藏目标的标注、关键点和引用只是不再产生可用跳转，不抛出未捕获异常；其它真正不存在的目标仍沿用现有诊断行为。
- 不删除媒体文件、不改变资料 ID、不改变资料挂靠顺序。

## 验收与测试

1. 合同测试确认 `visibility` 可解析、默认可见，并拒绝流程节点上的资料可见性字段。
2. Canvas Core V1/V2 测试确认隐藏资料不进入资源、关系、学习路径和索引可用快照，正常资料行为不变。
3. NodeChrome/GuideEditor 测试确认只有资料节点显示眼睛按钮，点击后在文档中持久化 `HIDDEN`，再次点击可以恢复。
4. LessonPage/LessonMap 测试确认隐藏资料不出现在当前步骤资料区、流程图或媒体跳转中。
5. 运行受影响包的定向测试、类型检查、Lint、构建和 `git diff --check`；检查最终 diff 只包含本功能文件与设计/计划文档。

## 非目标

- 不增加按角色、按工作区或按用户的资料权限模型。
- 不删除资料、不清理媒体资产、不重构既有附件挂靠模型。
- 不把“隐藏资料”扩展为隐藏整个指南；指南草稿/发布可见性继续由现有指南生命周期控制。
