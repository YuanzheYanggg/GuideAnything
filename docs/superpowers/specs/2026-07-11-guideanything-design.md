# GuideAnything 多模态教学画布设计

本设计以 [产品需求](../../PRD.md)、[架构说明](../../ARCHITECTURE.md)、[数据模型](../../DATA_MODEL.md) 和 [验收清单](../../ACCEPTANCE.md) 为规范来源。

## 方案比较

1. **React/Vite + React Flow + Fastify + SQLite（采用）**：成熟画布交互与清晰 API 边界能最快完成纵向切片；纯函数包承载引用算法，后续可单独优化。
2. **Next.js 全栈单体**：路由和部署简单，但弱化用户要求的前后端分离，并让画布/服务端边界更含混。
3. **自研 Canvas/WebGL + 独立 API**：超大图性能上限更高，但选择、端口、可访问性和编辑手势成本过高，不适合当前验收优先级。

## 设计决定

- 首个版本以可实际演示的 ERP 指南链路为中心，不加入实时协作、云存储或 AI 生成。
- 服务端工作副本持久化完整画布 JSON，并以 Zod schema 作为写入边界；关系表承担身份、版本、授权、搜索和媒体生命周期。
- 发布版本不可变，子指南固定到明确版本；展开采用确定性 ID 和来源追踪，折叠使用 `hidden`。
- 浏览器端使用受控 React Flow 与有限历史栈，API 使用乐观锁，冲突不覆盖。
- 自动化验证分为纯算法单测、Fastify 注入式 API 集成测试、React 组件测试和真实浏览器关键路径。

## 自检结果

- 无占位符、空白章节或未决产品选择。
- 权限、引用版本、坐标/ID、保存冲突与上传边界均有明确策略。
- 范围可按 M0-M4 独立验收，但各阶段共同形成同一条纵向产品链路。
