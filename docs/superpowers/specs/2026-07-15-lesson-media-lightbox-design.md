# 学习页图片与视频大屏预览设计

## 目标

在学习模式中，学习者点击图片或视频后可以打开一个清晰的大屏预览，便于查看界面截图和观看操作演示。Markdown 资料保持现有展示方式，不进入大屏预览范围。

## 范围

包含：

- 学习页图片点击预览
- 学习页视频点击预览
- 视频关键点在大屏播放器中继续可用
- 关闭按钮、遮罩点击关闭和 `Escape` 关闭
- 键盘可访问的弹窗语义和焦点入口
- 图片加载失败时沿用现有失败提示，不打开空预览

不包含：

- 编辑器画布中的媒体节点预览
- Markdown 大屏阅读器
- 数据结构、API 或媒体存储方式变更
- 图片缩放、旋转、下载等二级工具

## 交互设计

### 图片

学习页中的图片继续显示缩略内容。点击图片后打开预览弹窗：

- 使用 `contain` 保持原始比例
- 最大宽高不超过视口，并保留边距
- 显示图片替代文字和已有说明文字
- 点击关闭按钮、遮罩区域或按 `Escape` 关闭

### 视频

学习页中的视频继续显示原播放器。点击视频画面后打开预览弹窗：

- 使用原媒体地址和鉴权解析后的媒体源
- 保留原生播放、暂停、音量和进度控制
- 保留标题/说明文字
- 关键点按钮在弹窗内继续跳转播放位置，并通知当前教学步骤桥接逻辑
- 关闭弹窗后不改变当前步骤和学习进度

### 弹窗通用行为

- 弹窗使用 `role="dialog"` 和 `aria-modal="true"`
- 提供唯一的关闭按钮和可读的 `aria-label`
- 打开时焦点进入关闭按钮，关闭后焦点回到触发媒体
- 阻止背景页面滚动，但不改变 React Flow 或教学步骤状态
- 深色和浅色主题均使用现有主题变量
- 小屏幕下媒体仍以视口内最大尺寸展示，不产生横向溢出

## 组件与数据流

在学习页维护当前预览媒体状态：

```ts
type MediaPreview =
  | { kind: 'image'; node: CanvasNode<'image'>; source: string }
  | { kind: 'video'; node: CanvasNode<'video'>; source: string }
  | null;
```

学习页向当前步骤内容传递打开预览回调：

- `LessonImage` 接收 `onOpenPreview`，图片加载成功后点击触发
- `VideoNodeView` 接收可选的 `onOpenPreview`，保留编辑器中的现有调用兼容性
- 新增共享的 `MediaLightbox` 展示图片或视频，并接收视频关键点回调
- 预览媒体沿用 `useMediaSource` 的解析结果，避免为弹窗新增一套鉴权逻辑

预览状态只存在于学习页 React 状态中，不写回 `CanvasDocument`，也不调用保存接口。

## 错误与边界

- 媒体地址解析失败时不打开弹窗，继续显示现有的加载失败状态
- 图片或视频在弹窗内加载失败时显示媒体失败提示，并保留关闭能力
- 当前步骤切换时自动关闭预览，避免预览内容与步骤内容不一致
- 子指南学习页沿用同一套预览行为
- 编辑器中的 `VideoNodeView` 不传入预览回调，因此现有编辑行为不变

## 测试验收

新增或补充以下测试：

1. 学习页图片点击后出现媒体弹窗，显示图片和说明文字。
2. 学习页视频点击后出现媒体弹窗，播放器使用正确媒体源。
3. 弹窗内视频关键点仍能修改播放时间并触发步骤回调。
4. 点击关闭按钮、点击遮罩和按 `Escape` 都能关闭弹窗。
5. 弹窗打开期间不改变步骤索引、当前节点或保存状态。
6. 编辑器现有 `VideoNodeView` 测试继续通过。
7. Web 全量测试、类型检查和生产构建通过。

## 验收标准

- 学习者点击截图后，可以在当前页面直接看清大图。
- 学习者点击视频后，可以在更大播放器中观看并使用关键点。
- 不需要新开浏览器标签，不丢失当前教学步骤。
- 关闭预览后，学习页和步骤导航状态保持不变。
