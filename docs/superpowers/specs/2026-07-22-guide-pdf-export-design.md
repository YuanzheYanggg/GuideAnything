# GuideAnything 指南 PDF 对外导出设计

状态：设计已确认

日期：2026-07-22

## 1. 背景

GuideAnything 的流程画布运行在本地服务上。能够访问该服务的协作者可以查看画布，但外部接收者、手机用户或不使用该服务的人无法直接访问完整流程。

画布还包含不能简单通过一张截图表达的内容：

- 可展开的业务节点详情；
- Markdown 说明；
- 带编号点或矩形区域标注的图片；
- 带时间关键点的视频；
- 阶段、责任泳道、分支、连线标注和子指南。

本功能为当前已保存的指南生成一份适合外部阅读和转发的静态 PDF。PDF 不依赖接收者运行 GuideAnything；图片内容嵌入 PDF，公开视频通过二维码和公开地址访问。

## 2. 目标

### 2.1 第一版目标

- 从指南编辑器进入独立的 PDF 导出预览页；
- 用“流程总览 + 步骤详情 + 媒体附录”的方式组织内容；
- 保留现有流程阶段、责任泳道、连线、分支和连线标注；
- 无论编辑器中的详情节点当前是否展开，PDF 都输出完整详情；
- 图片在 PDF 中自包含，并保留图片标注及标注说明；
- 视频以静态视频卡片、说明、时间关键点、二维码和公开视频地址表达；
- 通过浏览器打印预览保存为 PDF；
- 在不改变画布数据模型、不新增 API 和数据库表的前提下完成第一版；
- 以自动化测试和浏览器渲染检查证明页面结构、媒体状态和打印布局可用。

### 2.2 非目标

- 第一版不生成 JPEG/PNG 导出；
- 第一版不把视频二进制嵌入 PDF；
- 第一版不提供脱离浏览器打印预览的一键二进制 PDF 下载器；
- 第一版不增加服务端异步导出任务、导出文件存储或导出历史；
- 第一版不创建公开的 GuideAnything 分享页面；
- 第一版不修改视频节点数据模型以强制增加封面字段；
- 第一版不改变原始画布、草稿历史、发布版本或媒体授权逻辑。

## 3. 已确认的产品决策

### 3.1 PDF 内容方案

采用“方案 B”：PDF 是一份适合外部阅读的流程指南，不是当前画布视口的截图。

内容顺序固定为：

1. 封面页；
2. 流程总览页；
3. 按流程步骤排列的详情页；
4. 图片标注资料页；
5. 视频资料卡。

总览页展示业务节点、阶段、责任泳道、分支、连线和连线标注。完整资源不全部挤进总览，而是在业务节点旁显示资源数量或类型提示，并在后续详情中展开。

### 3.2 视频链接

视频节点中的地址按长期稳定的公开 CDN/对象存储地址处理。二维码直接编码该地址，不需要登录 GuideAnything。

只有有效的 `http://` 或 `https://` 地址才能生成外部二维码。`/api/media/...` 这类需要当前服务和登录会话的地址不能作为外部视频链接；导出页必须明确显示警告，而不能生成误导性二维码。

视频没有专用封面时，第一版使用静态视频卡片，展示视频图标、标题、说明和关键点。后续可以单独增加视频封面字段或封面管理能力。

### 3.3 导出范围和版本

第一版从编辑器导出当前“已保存的草稿”。点击导出时若存在未保存修改，必须先保存成功；保存失败则不进入预览。

封面显示指南标题、摘要、标签、指南状态、当前 revision 和生成时间。已发布指南可以继续从编辑器导出当前草稿；发布版本专用导出不在第一版范围内。

默认只输出当前可见资料。资源节点标记为 `HIDDEN` 时，不进入总览资源统计或详情附录，避免把内部资料误发给外部人员。

### 3.4 生成方式

采用“独立导出预览页 + 浏览器打印/保存为 PDF”。导出页使用静态 React/HTML/CSS，不复用可编辑 React Flow DOM 的临时状态。

## 4. 用户流程

1. 作者或编辑者在指南编辑器顶部点击“导出 PDF”。
2. 编辑器检查保存状态。
3. 如有未保存修改，先执行已有的保存流程；保存失败时显示错误并停留在编辑器。
4. 保存成功后进入 `/guides/:guideId/export/pdf` 独立路由。
5. 导出页重新读取已保存的指南数据，并生成静态导出模型。
6. 导出页等待图片、二维码和其他可打印媒体准备完成。
7. 用户检查分页和警告，点击“打印 / 保存为 PDF”。
8. 浏览器打开打印预览，用户选择“保存为 PDF”。
9. 用户关闭预览或返回编辑器；导出过程不写回画布。

导出预览页只保留返回、状态提示、警告和打印操作，不显示节点拖拽控件、React Flow 缩放控制、缩略图、背景点阵、删除按钮、编辑器属性面板或编辑工具栏。

## 5. PDF 信息架构和布局

### 5.1 封面页

- 指南标题；
- 摘要和标签；
- 草稿/发布状态；
- revision；
- 生成时间；
- 图片、视频、Markdown 和步骤数量概览。

### 5.2 流程总览页

- 横向布局，优先保证流程方向和分支可读；
- 使用阶段背景和责任泳道标签表达业务上下文；
- 使用现有正交路由结果绘制连线；
- 保留业务连线标签；
- 主流程节点显示编号、类型、标题和简短摘要；
- 资料节点在总览中以资源数量/类型提示表示，完整内容进入步骤详情；
- 子指南引用显示为引用节点和固定版本信息，不在第一版强制展开为完整子指南图。

### 5.3 步骤详情页

按现有语义流程顺序排列。每个步骤包括：

- 步骤编号和标题；
- 所属阶段；
- 责任角色或系统；
- 节点完整详情；
- 分支条件和相关连线说明；
- 该步骤可见的 Markdown、图片和视频资源。

详情内容不依赖编辑器的 `detailExpanded` 临时状态，始终输出完整描述。

### 5.4 图片资料页

- 嵌入图片本身，不依赖接收者访问本地服务；
- 保持原始比例，不裁剪标注区域；
- 在图片上绘制 `POINT` 编号点或 `RECT` 区域；
- 按顺序显示标注编号、标题和说明；
- 显示标注关联的流程、Markdown、图片、视频或子指南；
- 标注补充图片按该标注的顺序继续展示。

### 5.5 视频资料卡

- 视频封面（如果第一版没有可用封面则使用静态视频卡）；
- 视频标题和说明；
- `keypoints` 的时间和标题；
- 本地生成的二维码；
- 可复制的公开视频地址。

二维码和文本链接属于同一视频卡片，避免二维码单独跨页。

### 5.6 打印布局

页面使用打印专用 CSS，隐藏预览操作栏并设置页边距、颜色保留和分页规则。流程总览采用横向内容布局，详情内容按纵向阅读流排列；媒体卡片和图片标注区块设置避免跨页拆分的规则。第一版以浏览器打印引擎为最终分页裁决，不额外引入 PDF 绘图库。

## 6. 技术架构

### 6.1 页面组件

预计新增和修改的主要文件：

```text
apps/web/src/App.tsx
apps/web/src/features/editor/GuideEditor.tsx
apps/web/src/features/export/GuidePdfExportPage.tsx
apps/web/src/features/export/GuidePdfExportDocument.tsx
apps/web/src/features/export/export-model.ts
apps/web/src/features/export/export-media.ts
apps/web/src/styles.css
apps/web/package.json
```

`GuidePdfExportPage` 负责加载数据、准备媒体、显示预览状态和调用 `window.print()`。`GuidePdfExportDocument` 只负责静态 PDF 内容渲染。编辑器只增加保存检查和导出入口，不把导出布局塞进 `GuideEditor`。

### 6.2 导出模型

`export-model.ts` 提供纯函数，将已保存的指南数据转换成 `GuidePdfExportModel`，包含封面、总览、步骤、媒体和警告。

导出模型复用已有领域逻辑：

- `deriveSemanticFlow` 确定步骤顺序；
- `routeCanvasEdges` 计算总览连线；
- 阶段和泳道数据生成上下文；
- 图片节点的 `annotations` 生成标注清单；
- 视频节点的 `keypoints` 生成时间点清单；
- `visibility`、`hidden`、来源追踪和资源挂靠关系决定是否展示。

模型转换是只读操作，不写回 `CanvasDocument`，不改变撤销栈，也不创建草稿历史记录。

### 6.3 媒体准备

- 受保护的 `/api/media/...` 图片通过现有 API 客户端获取为浏览器对象 URL；
- 外部图片地址直接用于预览，并等待加载完成；
- 视频公开地址不要求在预览页下载视频本体，只用于文本链接和二维码；
- 二维码在浏览器本地生成，不通过外部二维码服务发送视频 URL；
- 所有对象 URL 在导出页卸载时释放；
- 每个媒体资源有加载成功、失败和未准备状态。

第一版不要求把视频 URL 写入新的字段。视频节点已有“视频地址”编辑入口，作者可以填写公开 CDN/对象存储地址。

### 6.4 API 和数据模型边界

第一版不新增 API 路由、数据库表、合同 schema 或导出存储。导出页使用现有指南读取和媒体读取能力，并遵守当前鉴权。

导出预览页仍然需要作者/编辑者在本地服务中登录；生成出来的 PDF 和其中的图片内容不需要接收者登录。公开视频是否可访问由外部 CDN/对象存储的公开地址策略决定。

## 7. 错误处理、安全和降级

### 7.1 媒体错误

- 图片加载失败：保留标题、替代文字和说明，显示“图片载入失败”警告；
- 图片或补充图片加载失败：不阻塞其他页面渲染；
- 视频地址不是有效的 HTTP(S)：显示无法生成外部二维码的警告；
- 二维码生成失败：保留原始视频地址和关键点，不能隐藏失败原因；
- 没有流程节点：生成封面和资料附录，并显示“没有可展示的流程总览”；
- 资源全部隐藏：保留步骤主体，但显示没有可见资料。

预览页只有在已知资源准备状态后才启用打印按钮。即使用户选择继续打印，警告也必须出现在 PDF 中或预览顶部，避免静默丢失内容。

### 7.2 安全边界

- 不把 Vault 路径、SQLite 路径、Bridge token 或内部 locator 写入 PDF；
- 二维码只编码作者明确提供的公开 HTTP(S) 视频地址；
- 受保护的 `/api/media/...` 地址不作为外部二维码目标；
- 导出页沿用现有指南权限，不能通过路由绕过编辑者/作者授权；
- Markdown 继续使用现有安全渲染链路，不执行原始 HTML 或脚本；
- 导出不把外部媒体转发到第三方二维码服务。

## 8. 测试和验收

### 8.1 单元测试

- 导出模型按语义流程顺序排列步骤；
- 隐藏资源不进入总览统计或详情；
- 详情输出与 `detailExpanded` 展示状态无关；
- 图片标注按 `order` 生成，点和矩形区域内容完整；
- 标注标题、正文、补充图片和关联节点映射正确；
- 视频关键点按时间和原有顺序展示；
- 有效公开 URL 可以生成二维码输入；
- `/api/media/...` 和无效协议产生警告；
- 空流程、缺失节点和媒体失败路径可生成可读的降级模型。

### 8.2 组件测试

- 编辑器显示“导出 PDF”入口；
- 未保存状态先保存，保存失败不跳转；
- 导出页显示封面、总览、步骤详情、图片标注和视频卡片；
- 预览页隐藏编辑器控件；
- 加载中时打印按钮不可用；
- 失败媒体显示可见警告；
- 打印操作调用浏览器打印入口。

### 8.3 浏览器验收

使用包含流程、判断、详情、阶段/泳道、图片标注和公开视频地址的真实指南验证：

1. 进入编辑器并制造一项未保存修改；
2. 点击导出，确认先保存再进入预览；
3. 确认预览页没有拖拽、属性面板、缩放控件或编辑按钮；
4. 确认总览连线、分支和泳道没有丢失；
5. 确认完整详情和图片标注出现在后续页面；
6. 确认视频卡有关键点、二维码和公开链接；
7. 扫描二维码，确认跳转到预期公开视频地址；
8. 进入打印预览并保存 PDF；
9. 检查最终 PDF 没有节点重叠、分页截断、空白图片或二维码遮挡。

验证命令至少包括：

```bash
pnpm --filter @guideanything/web test
pnpm --filter @guideanything/web typecheck
pnpm --filter @guideanything/web build
git diff --check
```

## 9. 后续扩展

第一版完成并验证后，可以在不改变导出模型的前提下增加：

- JPEG/PNG 高清长图导出；
- 视频封面字段或自动生成封面；
- 直接下载 PDF 文件，减少打印预览步骤；
- 导出 ZIP/HTML，使视频、图片和流程在离线环境中一起分发；
- 已发布不可变版本的专用 PDF 导出；
- 生成可长期访问的公开分享页。
