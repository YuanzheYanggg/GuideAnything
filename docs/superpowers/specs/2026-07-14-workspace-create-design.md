# 新建工作区设计

## 目标

让作者可以从 `/workspaces` 创建工作区；创建成功后进入新工作区，编辑者和学习者不显示创建入口。

## 方案

- 前端目录页在作者角色下显示“新建工作区”按钮。
- 使用现有 modal 视觉样式展示表单，字段为名称、slug、描述、图标和颜色。
- 通过现有 `POST /api/workspaces` 创建，复用后端的作者权限和 Zod 校验。
- 创建成功后刷新工作区列表并导航到新工作区；失败时在表单内显示服务端错误。
- 不改变已有工作区数据、指南创建流程或编辑者/学习者权限。

## 验证

- 前端测试覆盖作者入口、表单提交、成功导航、非作者隐藏入口和错误展示。
- 后端现有创建接口测试继续作为权限与持久化保障。
- 运行工作区前端测试、API 工作区测试、类型检查和构建。
