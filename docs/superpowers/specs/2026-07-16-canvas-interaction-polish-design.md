# 画布层级、节点明细与精确连线交互设计

## 目标

完善 GuideAnything 编辑画布的三个高频操作：

1. 作者可以删除不再需要的业务阶段或责任泳道，而不会误删流程节点、业务连线或教学步骤。
2. 作者编辑节点明细时获得一个完整、不溢出节点边界的编辑框；读图时节点保持紧凑，按需展开完整明细。
3. 作者可以像协作白板一样调整一条业务连线的两端、边缘落点和视觉样式，并在保存、刷新和学习页中保持结果。

本设计修改编辑画布、学习页的只读连线渲染和画布文档契约。不会改变 API 路由、数据库表、已有学习路径顺序或资料挂靠关系。

## 当前问题与根因

- `HierarchyPanel` 和 `GuideEditor` 只提供阶段/泳道的新增、改名与排序，没有删除回调或文档更新逻辑。
- `InlineNodeTextEditor` 在双击节点明细时把节点正文替换为三行 `textarea`。普通流程节点在 React Flow 中被显式设置为 `240 x 104`，而节点内容允许溢出，因此输入框越过了节点边界。
- `CanvasEdge` 只保存源/目标、端口和标注；编辑器为所有新连线写入同一份蓝色 `2px` 单向箭头样式。React Flow 支持重连，但编辑器没有提供 `onReconnect` 持久化回调，节点也只有少数固定端口。

## 范围与边界

### 包含

- 删除阶段与泳道，解除相关节点归属。
- 节点明细的实体编辑框、单行摘要、展开和收起。
- 可持久化的业务连线颜色、粗细、线型、箭头方向、两端精确锚点和重新连接。
- 旧画布数据向后兼容。
- 单元、组件、集成和浏览器验收。

### 不包含

- 删除节点的确认策略变更。
- 资料挂靠虚线和子指南展开派生线的样式/重连编辑。
- 手工添加、删除或拖动中间折点；折线路由仍由现有避障算法计算。
- 自定义任意 CSS、任意 SVG 箭头或第三方白板依赖。

## 方案选择

已选择精确锚点方案，而非固定端口或仅自动选边方案。每条业务连线会保存起点和终点在节点四边上的精确相对位置；渲染层根据该位置构建正交避障路径。这样可满足“任意一边、任意位置”的要求，且不会把视觉交互限制为四个中点。

## 用户交互

### 阶段与泳道删除

每一行阶段/泳道管理项新增删除按钮。点击后出现确认对话框，包含名称和将被解除归属的节点数量。

- 删除阶段：移除该阶段，所有 `stageId` 相符的一级主流程节点清除 `stageId`，并显示在“未分阶段”。
- 删除泳道：移除该泳道，所有 `laneId` 相符的一级主流程节点清除 `laneId`，并显示在“未分配责任”。
- 不删除节点、业务边、资料挂靠、教学步骤、入口或出口节点。
- 剩余阶段/泳道重新编号 `order`，并通过既有历史栈支持撤销。
- 自动整理预览激活时保持管理器锁定，不允许删除。

### 节点明细

双击流程节点的明细区域时，打开一个实体 `role="dialog"` 编辑框。它以实色表面覆盖在画布之上，不把输入控件嵌进固定尺寸节点。

- 内容为多行 `textarea`，提供“保存”和“取消”；`Escape` 取消，`Meta/Ctrl + Enter` 保存。
- 打开时焦点进入文本框；关闭后焦点返回原节点的“详情”触发器。
- 保存后的节点默认只显示第一行摘要；空明细继续显示当前的引导占位文字。
- 非空明细在右下角显示“详情”。点击后临时扩展节点至完整内容所需高度，按钮变为“收起”；收起后恢复保存的手工尺寸或默认紧凑尺寸。
- 展开状态是编辑器本地展示状态，不写入 `CanvasDocument`，也不会让自动测量出的展开高度覆盖作者手工调整的尺寸。
- 展开状态下隐藏尺寸拖拽控制，避免把临时自动高度误保存为用户尺寸。

### 业务连线

选中一条可编辑业务连线时，在最长可见线段附近展示无背景穿透的横向工具栏：

- 颜色：产品预设色板（默认主题强调色、蓝、绿、黄、红、紫）。
- 粗细：`1 / 2 / 3 / 4` 像素。
- 线型：实线、虚线、点线。
- 箭头：无箭头、正向、反向、双向。
- 两端：显示可拖拽控制点；拖到任意节点的上、右、下、左边缘任意位置即可重新连接。

每个节点渲染连续的四边连接面，用于新建连线和重连落点。已有连线按自身锚点渲染专用物理端口，确保同一节点的多条线能停在不同位置。选择或拖拽不改变判断节点已有的“是/否”业务语义。

资料挂靠线和子指南派生线继续使用现有虚线样式，保持不可选择、不可重连、不可修改样式。

## 数据与渲染架构

### 文档契约

扩展 `CanvasEdge` 的可选 `presentation` 字段：

```ts
type EdgeAnchor = {
  side: 'TOP' | 'RIGHT' | 'BOTTOM' | 'LEFT';
  offset: number; // 0 到 1，沿该边从左到右或从上到下
};

type EdgePresentation = {
  color?: 'default' | 'blue' | 'green' | 'yellow' | 'red' | 'purple';
  width?: 1 | 2 | 3 | 4;
  pattern?: 'solid' | 'dashed' | 'dotted';
  arrows?: 'none' | 'forward' | 'reverse' | 'both';
  sourceAnchor?: EdgeAnchor;
  targetAnchor?: EdgeAnchor;
};
```

字段全部可选。旧连线没有 `presentation` 时使用现有视觉默认值：accent 蓝、`2px`、实线、正向箭头，并按旧端口计算锚点。

`sourceHandle` 和 `targetHandle` 继续保存业务端口语义，例如判断节点的 `yes`、`no`。渲染层生成的专用物理端口 ID 不写回文档；因此重连和精确落点不会破坏分支标签、知识流构建或已有数据。

使用预设颜色和受限枚举，而非接收任意 CSS 字符串；所有新字段由 Zod 契约验证。

### 编辑器状态与组件职责

- `GuideEditor`：维护本地展开节点集合、待确认删除对象、选中的业务边、正在重连端点和连接落点；负责把所有文档变更经 `commit` 写入历史与保存状态。
- `HierarchyPanel`：展示删除按钮和影响数量，通过显式回调请求删除；不直接修改文档。
- `NodeDetailDialog`：处理节点明细草稿、焦点、保存和取消；只通过回调更新描述。
- `FlowNode` / `NodeChrome`：渲染一行摘要、详情展开控制和连续边缘连接面；接收展示状态，不持久化该状态。
- `EdgeToolbar`：只服务于可编辑业务边，调用单一的 `updateEdgePresentation` 回调。
- `OrthogonalEdge`：将样式和箭头映射为 SVG 属性，并把精确锚点传给正交路由；仍使用既有避障与标签定位规则。
- `canvas-core/routing`：以可选锚点替代默认端口中点作为路径端点，保持阶段间、回流和分支路径分类。
- `LessonPage`：只读地复用已持久化的业务边样式、箭头和锚点；不显示编辑工具栏或重连控制点。

## 状态转换与失败处理

```text
删除阶段/泳道
  -> 确认影响节点数量
  -> 移除层级对象 + 清除相应归属 + 重新编号
  -> commit + 历史栈

新建/重连业务边
  -> 记录指针在源端和目标端的边缘锚点
  -> 校验连接目标
  -> 更新端点、语义端口和 presentation 锚点
  -> 路由 + commit
```

- 无效重连（落在画布空白或不可连接对象）保留原端点和锚点，不写入文档。
- 自动整理预览、派生边、资料挂靠边和来源追踪边不暴露编辑操作。
- 删除确认、保存失败或重连失败都不改变当前已保存文档；既有自动保存错误反馈继续复用。
- 组件只向 React 输出文本，不引入 HTML 注入或未经校验的 SVG/CSS 值。

## 验收与验证

### 自动测试

1. 删除阶段/泳道后，只解除对应节点归属，连线、教学步骤和节点数量不变；撤销后恢复。
2. 节点明细对话框双击打开、取消不保存、保存后单行截断、详情展开/收起不改变文档尺寸。
3. 旧 `CanvasEdge` 仍可解析并显示默认样式；新样式、箭头和锚点通过契约验证与 round-trip。
4. 新建和重连边会更新真实端点与精确锚点；无效重连不改变原边。
5. 路由使用锚点作为端点，仍保留正交避障和回流分类。

### 浏览器验收

在 `http://127.0.0.1:5175` 的独立 worktree 栈完成：

1. 创建阶段和泳道、为节点分配归属、删除两者并确认节点回到未分配区域。
2. 输入多行节点明细，验证实体编辑框无漏底、保存后单行显示、详情展开和收起。
3. 选择连线，逐项切换四类样式；拖动两端到不同节点边缘位置，保存、刷新后检查样式与落点保留。
4. 打开对应学习页，检查业务边沿用已保存的样式和锚点，但没有编辑控制点。
5. 检查桌面和窄视口，不出现裁剪、横向溢出、Vite 错误遮罩或相关控制台警告/错误。

自动验证使用应用内 web 包的 Vitest、TypeScript、构建、`git diff --check`，再以浏览器 DOM、截图与控制台作为最终证据。
