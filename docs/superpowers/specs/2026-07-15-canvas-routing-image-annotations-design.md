# GuideAnything 画布走线与图片讲解设计

> 日期：2026-07-15  
> 状态：已实现并完成运行验收  
> 范围：自动布局与业务走线、图片快速标注、镜头式播放、标注关联资料

## 1. 背景与目标

当前自动整理已经能按入口、阶段、责任泳道和资料挂靠关系放置节点，但主流程仍主要依赖固定网格和 React Flow `smoothstep` 边。判断分支、合流、跨阶段边和循环回流同时出现时，业务顺序与固定位置约束会冲突，造成连线交叉、重叠、穿越节点或从节点不合理方向出入。

图片目前只能保存媒体地址、替代文字和说明。学习者可以在灯箱中查看图片，却不能按顺序聚焦图片局部，也不能从图片中的讲解点返回对应业务资料或其他节点。

本设计达成以下结果：

1. 业务流程在同一阶段内从左到右、阶段之间从上到下排列。
2. 判断分支、合流和循环有稳定规则，业务线采用正交路径并尽量减少交叉和重叠。
3. 作者可在图片上快速创建编号点或矩形区域，并可选保存局部镜头。
4. 学习者可按标注顺序播放图片讲解，并从讲解卡片进入关联资料或节点。
5. 业务流程仍是学习模式的唯一结构来源；图片标注不会形成第二套教学步骤。

## 2. 范围与非目标

### 2.1 本轮范围

- 二维、业务约束的自动布局。
- 判断分支、合流、跨阶段和循环回流的布局规则。
- 自动业务边的正交通道和自定义边渲染。
- 图片点标注、矩形标注、顺序调整和可选镜头保存。
- 图片讲解的上一个、下一个、编号跳转和可选自动播放。
- 标注关联当前指南内的业务节点、资料节点或子指南节点。
- 编辑器、发布快照和学习模式的端到端兼容。
- 协议、算法、交互与可访问性测试。

### 2.2 非目标

- 不引入通用图布局引擎或新的大型第三方依赖。
- 不承诺任意有向图在数学意义上零交叉。
- 不实现 AI 图片识别、OCR、自动生成标注或语音讲解。
- 不实现评论、审核流、实时协作或演示录制。
- 不实现跨指南任意深链；关联目标必须属于当前发布快照。
- 不重构现有阶段、泳道、子指南或教学步骤模型。

## 3. 用户故事

### 3.1 流程作者

作为流程作者，我希望自动整理后主线、分支和回流具有固定视觉语法，从而无需手工拖动大量节点和连线才能看懂流程。

作为流程作者，我希望点击截图即可添加讲解点，并可在需要时保存局部放大镜头，从而快速制作可播放的操作说明。

作为流程作者，我希望标注能够关联对应说明、视频、另一张图或业务节点，从而避免把解释性关系画成更多业务线。

### 3.2 学习者

作为学习者，我希望像浏览幻灯片一样依次查看截图中的关键位置，从而不必在一张复杂截图中自行寻找操作区域。

作为学习者，我希望从标注旁直接打开相关资料，并能返回原图片和原标注位置，从而保持学习上下文。

## 4. 自动布局设计

### 4.1 布局优先级

布局约束按以下顺序执行：

1. 业务拓扑和入口到出口的顺序。
2. 阶段顺序及阶段边界。
3. 判断分支顺序和合流位置。
4. 责任泳道信息。
5. 尽量保留作者已有的相对顺序。

责任泳道继续作为责任语境和布局提示，但不得迫使主流程产生明显折返。若责任列与业务顺序冲突，以业务顺序为准；节点仍显示责任徽标，阶段/泳道面板保留原有编辑能力。

### 4.2 二维放置规则

- 阶段整体按 `FlowStage.order` 从上到下排列。
- 每个阶段内部按拓扑 rank 从左到右放置主流程节点。
- 同 rank 节点按判断分支优先级、已有纵坐标、节点 ID 稳定排序。
- 阶段内节点超过可读宽度或分支需要独立空间时，可创建阶段内部子行；子行仍保持从左到右，不采用会导致方向反转的蛇形排列。
- 新阶段从新的左侧基线开始，跨阶段边向下进入下一阶段。
- 合流节点 rank 必须晚于全部可达前置分支。
- 未连通主流程放入独立的“未连接”区域，不混入入口主链。
- 资料节点继续放在所属主流程节点的详情区域，不参与业务拓扑排名。
- 展开子指南产生的来源节点不参与宿主自动布局。

### 4.3 判断与回流规则

- 判断节点的主路径优先向右。
- `yes` 或语义上的继续路径靠近主干；`no`、异常、驳回路径优先向下进入分支子行。
- 未标注分支使用稳定的边 ID 顺序，避免重复整理时跳动。
- 合流节点放在所有分支之后，并尽量与主路径对齐。
- 指向相同或更早 rank 的边视为回流边。
- 回流边统一走当前阶段或相关阶段的外侧回流通道，不穿越主流程主体。
- 自环和强连通循环必须出现在布局诊断中；布局保持确定性，不尝试改变业务拓扑。

### 4.4 正交路由

自动整理为每条可见、真实业务边计算展示路由：

- 路由只由水平和垂直线段组成，并使用小圆角连接。
- 正向边默认从右侧输出、从左侧输入。
- 下行分支可从底部输出，从目标顶部或左侧输入。
- 跨阶段边通过阶段之间的水平间隔通道下行。
- 回流边通过阶段外侧通道绕行。
- 多条边使用同一逻辑通道时分配固定间距的平行偏移，不完全重叠。
- 路由不得与节点矩形相交；检测到相交时向相邻空闲通道偏移。
- 资料挂靠边仍是不可编辑的细虚线，并在所属节点附近采用短路径。

路由是由节点位置和边关系派生的展示数据，不写入 `CanvasDocument`。作者移动节点后，未重新自动整理前使用安全的正交回退路径；应用自动整理后重新计算完整通道。

### 4.5 布局预览与诊断

现有“预览后应用”工作流保留。报告新增：

- 无法避开的节点碰撞数量及相关边 ID；
- 发生节点避障偏移的边；
- 回流边和循环节点；
- 未连通主流程；
- 发生通道偏移的边。

预览不保存、不进入撤销历史；应用后写入节点位置，并可用撤销恢复。边路由作为派生数据不进入撤销历史。

### 4.6 验收标准

- 单主线、树形判断分支、菱形分支合流不出现业务边交叉。
- 业务边不穿过任何节点主体。
- 多条共线边可并行显示，不完全覆盖。
- 跨阶段边优先走阶段间隙。
- 循环回流走阶段外侧，不横穿主流程。
- 相同输入文档重复布局产生相同节点位置、诊断和路由。
- 旧文档无阶段、无泳道或含循环时仍可完成布局，不丢失节点和边。

## 5. 图片标注数据模型

图片节点新增可选 `annotations` 数组。旧文档不含该字段时等价于空数组。

```ts
interface ImageAnnotation {
  id: string;
  order: number;
  title: string;
  body?: string;
  shape: 'POINT' | 'RECT';
  region: {
    x: number;
    y: number;
    width?: number;
    height?: number;
  };
  camera?: {
    centerX: number;
    centerY: number;
    zoom: number;
  };
  targetNodeId?: string;
}
```

坐标和尺寸均使用相对图片原始内容区域的 `0..1` 归一化数值，避免节点缩放、响应式布局或图片分辨率变化导致标注漂移。

约束：

- 每张图片最多 500 个标注。
- `title` 必填，最长 200 字；`body` 最长 5,000 字。
- `order` 为非负整数，保存时归一化为连续顺序。
- 点标注只使用 `x/y`；矩形标注还需正数 `width/height`，且区域不得超出图片。
- `zoom` 限制在 `1..8`。
- `targetNodeId` 可指向当前文档中的一级主流程、资料或子指南节点，但不得指向自身。
- 目标删除后允许保留标注和失效 ID，以支持恢复与明确提示；因此图片关联不作为 `CanvasDocumentSchema` 的硬引用完整性错误。
- 发布检查会把失效关联列为警告，但不阻断保存或发布。

## 6. 图片标注编辑体验

### 6.1 进入标注模式

作者选中图片节点后，可在属性面板点击“编辑图片标注”。标注模式在专用图片编辑层中打开，避免在缩放画布上误触节点拖拽或连线。

### 6.2 快速创建

- 单击图片空白处创建点标注。
- 选择矩形工具后拖拽创建区域标注。
- 新标注获得下一个顺序编号，并立即打开紧凑编辑卡片。
- 作者可输入标题、说明和关联目标。
- 作者可选择缩放倍率并点击“保存当前镜头”；镜头中心取当前标注中心，未保存镜头的标注播放时使用自动适配区域。
- 标注列表支持上移、下移和删除，不实现本轮范围外的自由拖拽排序。

### 6.3 关联目标

目标选择器优先展示：

1. 图片所属业务节点；
2. 挂靠在同一业务节点的 Markdown、图片和视频；
3. 当前指南中的其他主流程节点和子指南节点；
4. 当前指南中的其他资料节点。

关联关系不渲染成画布业务边。图片节点显示标注数和有效关联数；被引用节点显示轻量徽标，不改变节点拓扑。

## 7. 图片讲解播放

### 7.1 播放入口

- 学习模式中的图片资料卡和灯箱提供“开始讲解”。
- 无标注图片继续使用当前普通灯箱。

### 7.2 播放行为

- 初始显示完整图片和标注总数。
- 开始后按 `order` 聚焦第一个标注。
- 有 `camera` 时使用保存的中心点和缩放；无 `camera` 时根据点或矩形区域自动计算合适镜头。
- 当前编号点或区域高亮，其余标注降低视觉权重。
- 旁侧讲解卡显示编号、标题、说明、进度和关联资料按钮。
- 支持上一个、下一个、直接点击编号、左右方向键和可选自动播放。
- 自动播放默认关闭；开启后每个标注停留固定时长，用户手动交互后暂停。
- 最后一项之后停止，不自动跳到下一业务教学步骤。
- 系统开启 `prefers-reduced-motion` 时立即切换镜头，不执行平移动画。

### 7.3 关联目标导航

- 关联资料属于当前步骤时，在学习侧栏或媒体灯箱中打开。
- 关联另一业务节点时，先显示节点摘要，作者可继续切换到对应教学步骤；若没有显式教学步骤，则聚焦只读流程画布。
- 关联子指南时先显示固定版本摘要，确认后沿用现有子指南打开逻辑。
- Markdown、图片和视频资料预览返回后恢复原图片、标注索引和镜头状态；切换教学步骤或进入子指南属于显式上下文切换。
- 目标不存在时按钮显示“关联资料已失效”并禁用，不抛出运行时错误。

## 8. 组件和数据流

### 8.1 Contracts

`packages/contracts/src/canvas.ts` 增加图片标注 schema，并扩展图片节点 `data`。现有 `schemaVersion: 1` 保持不变，因为新增字段可选且旧快照仍可解析。

### 8.2 Canvas Core

`packages/canvas-core` 保留纯函数边界：

- 层级布局负责阶段、rank、分支、合流和子行位置。
- 新的路由模块接收节点矩形与业务边，返回按边 ID 索引的正交路径及诊断。
- 路由模块不修改 `CanvasDocument`。
- 图片标注排序和目标有效性检查使用独立纯函数，供编辑器与发布检查复用。

### 8.3 Web 编辑器

- `GuideEditor` 在自动整理预览和正常画布中计算路由结果。
- 真实业务边使用自定义正交边组件；资料挂靠边继续使用单独样式。
- `ImageNode` 只显示标注摘要，不在普通画布节点尺寸内直接编辑标注。
- 新增图片标注编辑器，管理工具、视窗、标注选择和属性表单。
- 所有标注修改通过现有 `commit`/历史栈写入，支持撤销和重做。

### 8.4 学习模式

- `MediaLightbox` 扩展为支持图片讲解播放器。
- 图片播放器管理当前标注、镜头、自动播放和返回状态。
- `LessonPage` 负责把关联节点解析为当前步骤、资料预览、画布聚焦或子指南打开动作。

## 9. 错误与兼容处理

- 图片加载失败时保留现有错误状态，禁用标注播放。
- 标注坐标异常的旧数据在渲染时夹取到 `0..1`，保存前由 schema 拒绝新异常数据。
- 空标注数组不显示播放按钮。
- 标注目标失效不导致文档无法载入、保存或发布。
- 自动布局遇到循环、孤立节点或无法完全避开的交叉时返回诊断并仍提供可应用预览。
- 路由计算失败时回退到现有 `smoothstep`，不得阻断编辑器载入。
- 当前未提交的媒体灯箱改动视为现有工作基础，实施不得覆盖或撤销这些改动。

## 10. 安全、性能与可访问性

- 标注标题和说明作为纯文本渲染，不使用不受控 HTML。
- 关联只保存节点 ID，不复制敏感资料内容或生成外部 URL。
- 布局和路由只处理可见一级业务节点与真实业务边，来源节点和隐藏边不参与宿主计算。
- 路由结果按节点位置和边关系 memoize；拖动期间使用轻量回退路径，拖动结束后再完整计算。
- 标注点和工具按钮提供可访问名称；标注列表、上一个/下一个和目标按钮支持键盘操作。
- 播放器使用对话框语义、焦点管理、Escape 关闭和关闭后焦点恢复。
- 颜色不是唯一标识；当前标注同时使用编号、边框和讲解卡状态。

## 11. 测试与验证

### 11.1 协议测试

- 旧图片节点不含 `annotations` 时通过。
- 合法点、矩形、镜头和关联目标通过。
- 越界坐标、非法尺寸、非法缩放、重复 ID 和超限数量失败。

### 11.2 布局与路由测试

- 主线、树形分支、分支合流、跨阶段、循环回流和多边共用通道。
- 节点矩形碰撞检测。
- 确定性和输入文档不变性。
- 旧文档、无阶段、无泳道、孤立节点和展开子指南兼容。
- 1,000 节点性能用例继续通过，并为路由增加合理的性能预算。

### 11.3 编辑器测试

- 创建点/矩形、重排和删除标注。
- 保存当前镜头与清除镜头。
- 选择有效目标和显示失效目标。
- 撤销/重做标注修改。
- 应用自动整理后使用正交边，取消预览不改变文档。

### 11.4 学习模式测试

- 普通图片灯箱保持兼容。
- 按顺序播放、编号跳转、键盘导航和自动播放暂停。
- 有镜头、无镜头和减少动态效果。
- 打开关联资料、切换关联步骤、打开子指南和返回状态恢复。
- 失效关联安全降级。

### 11.5 最终验证

- 运行受影响包的单元测试和类型检查。
- 运行 Web build。
- 启动本地 Web/API，使用包含主线、判断、合流、循环和图片标注的演示文档进行视觉检查。
- 检查宽屏与窄屏灯箱、焦点、溢出、裁切和动画降级。
- 最终审查 diff，确保未覆盖现有媒体灯箱工作或引入无关改动。

## 12. 后续能力

后续按价值优先考虑：局部自动整理、节点锁定、发布前演示路径预览、发布完整性检查、版本差异和全局搜索定位。激光笔、临时涂画、录音时间轴、AI 自动标注和导出离线演示包不进入当前实施计划。
