# GuideAnything 节点内文字编辑设计

> 日期：2026-07-15  
> 状态：待实施  
> 范围：编辑画布中的节点文字原位编辑

## 1. 目标

作者无需移动到右侧属性面板，即可直接在节点内部修改日常文字内容。节点内编辑必须沿用现有文档提交、撤销、重做、草稿保存和协议校验链路，不能把编辑回调或临时输入状态写入 `CanvasDocument`。

## 2. 支持范围

- 开始、结束、流程、判断和数据节点：编辑标题；已有节点明细时也可原位编辑明细。
- Markdown 节点：原位编辑原始 Markdown，退出编辑后恢复渲染预览。
- 图片节点：原位编辑图片说明。
- 视频节点：原位编辑视频说明。
- 固定版本子指南标题保持只读，避免在宿主指南中制造与来源版本不一致的标题。
- 图片替代文字、媒体地址、上传、图片标注、视频关键点、阶段、泳道和资料挂靠继续由右侧属性面板编辑。

右侧面板保留现有文字字段作为兼容入口，但节点内编辑是日常文字修改的主要入口。本轮不删除字段，避免缩减键盘用户和批量检查时的备用路径。

## 3. 交互规则

### 3.1 进入与显示

- 双击可编辑文字进入原位编辑。
- 可编辑文字在节点选中或悬停时显示轻量编辑提示，但不增加常驻按钮。
- 单行标题使用单行输入框；明细、Markdown 和媒体说明使用多行文本框。
- 进入编辑后自动聚焦；单行标题默认选中全部文字，多行内容保留光标定位能力。
- 空的可选字段在节点选中后显示可双击的占位文案，以便作者直接补充内容。

### 3.2 保存与取消

- 单行标题按 `Enter` 保存。
- 多行内容按 `Cmd/Ctrl + Enter` 保存，普通 `Enter` 插入换行。
- `Esc` 取消并恢复进入编辑前的文字。
- 点击节点外部或输入框失焦时保存。
- 标题去除首尾空白后不能为空；空标题不保存，并继续停留在编辑状态。
- 可选明细或说明允许清空；清空后从持久化数据中删除对应可选字段。
- Markdown 允许空字符串，与现有协议保持一致。

### 3.3 画布事件隔离

- 输入框使用 React Flow 的 `nodrag`、`nopan` 和 `nowheel` 事件隔离类，编辑文字不会拖动节点、平移画布或触发缩放。
- 双击文字不得触发连线、节点展开或媒体播放。
- 自动整理预览期间所有节点内编辑入口关闭。
- 图片标注编辑器、连线标注编辑器和节点文字编辑不能同时激活；进入新的编辑器前先提交当前有效文字。

## 4. 组件与数据流

新增一个只服务编辑器的节点编辑上下文，向节点组件提供以下能力：

```ts
interface InlineNodeEditing {
  enabled: boolean;
  updateText(nodeId: string, field: InlineTextField, value: string): void;
}

type InlineTextField =
  | 'label'
  | 'description'
  | 'markdown'
  | 'imageCaption'
  | 'videoCaption';
```

数据流如下：

1. `GuideEditor` 在 React Flow 外提供编辑上下文。
2. 节点组件只在本地保存当前输入草稿和是否正在编辑，不修改 `node.data`。
3. 用户确认后，节点组件通过上下文提交 `nodeId + field + value`。
4. `GuideEditor` 根据节点类型校验字段，构造新的 `CanvasNode`，复用现有 `commit` 写回文档。
5. `commit` 继续负责协议校验、撤销历史、React Flow 节点同步和“未保存”状态。
6. 保存草稿仍只序列化纯数据，不包含上下文、回调或临时输入状态。

节点展示组件在学习页面或其他只读环境中没有编辑上下文时保持当前只读行为，不需要额外配置。

## 5. 历史与并发行为

- 一次确认只调用一次 `commit`，生成一条撤销记录；输入过程不逐字写入历史。
- 取消编辑不改变文档，也不生成历史记录。
- 右侧面板在节点内输入期间仍显示最后一次已提交值；节点内确认后两处同步更新。
- 若节点在编辑期间被删除或文档被外部替换，本地草稿直接丢弃，不重新创建节点。
- 应用自动整理只改变位置；已经确认的文字保留，尚未确认的无效空标题不会自动提交。

## 6. 可访问性

- 输入框具有包含节点名称和字段名称的 `aria-label`。
- 可编辑展示元素支持键盘进入：节点选中后，`Enter` 或 `F2` 编辑主要标题；Markdown、图片和视频节点编辑其主要文字内容。
- 编辑状态通过输入框本身和清晰焦点样式表达，不只依赖颜色。
- 保存失败或标题为空时使用就地错误提示，并通过 `aria-live` 宣告。

## 7. 测试与验收

### 7.1 单元与组件测试

- 双击流程标题进入输入框，修改并按 `Enter` 后保存到草稿载荷。
- `Esc` 取消，不产生文档变更或保存载荷变化。
- 标题为空时不提交，并显示错误。
- 多行说明的普通 `Enter` 保留换行，`Cmd/Ctrl + Enter` 才提交。
- Markdown 提交后恢复安全渲染预览。
- 图片和视频说明允许清空，并从可选字段中移除。
- 固定版本子指南不出现原位编辑入口。
- 自动整理预览期间双击不进入编辑。
- 一次确认对应一次撤销，撤销后恢复原文字。
- 学习页面继续只读，节点双击不会出现输入框。

### 7.2 浏览器验收

在隔离 QA 服务中完成以下真实操作：

1. 双击流程标题，修改并按 `Enter`。
2. 双击 Markdown，输入多行内容并按 `Cmd/Ctrl + Enter`。
3. 保存草稿、刷新页面，确认两项内容保持。
4. 撤销一次，确认只撤销最后一次文字提交。
5. 进入自动整理预览，确认无法误触进入文字编辑。
6. 检查编辑过程中节点没有移动，控制台没有 warning 或 error。

### 7.3 完成标准

- 支持范围中的日常文字均可直接在节点内完成修改。
- 保存、刷新、撤销、重做和自动整理不会丢失已确认文字。
- 临时输入状态和函数不会进入文档或 API 载荷。
- 现有右侧属性编辑、图片标注、视频关键点、节点拖拽和连线行为无回归。
- `pnpm lint`、`pnpm test`、`pnpm typecheck`、`pnpm build` 和 `git diff --check` 全部通过。

## 8. 非目标

- 不实现富文本所见即所得编辑器。
- 不实现多人实时协同编辑、冲突合并或字段锁。
- 不支持在节点内修改媒体地址、上传文件、阶段、泳道、挂靠和子指南来源。
- 不删除右侧属性面板中的兼容文字字段。
- 不改变 `CanvasDocument` 协议或发布快照结构。
